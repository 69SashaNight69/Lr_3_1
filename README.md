## Опис

Цей проект призначений для ознайомлення з підходами балансування задач за допомогою `Fork/Join Framework` в Java. Програма вирішує задачу множення двох матриць, розділяючи її на підзадачі та виконуючи їх паралельно за допомогою двох різних підходів до балансування задач:

1. **Work Stealing** — підхід, при якому потоки можуть "вкрадати" роботу один в одного для досягнення балансування навантаження.
2. **Work Dealing** — підхід, при якому кожен потік отримує свою частину роботи без перерозподілу.

Програма генерує дві випадкові матриці заданого розміру, а потім множить їх за допомогою обох підходів. Заміряється час виконання кожного з підходів для порівняння продуктивності.

## Основні функціональності

- Генерація випадкових значень для матриць з заданими розмірами та діапазоном чисел.
- Виконання множення матриць за допомогою двох підходів паралельної обробки:
  - **Work Stealing** через `Fork/Join Framework`.
  - **Work Dealing** через `ExecutorService` з пулом потоків.
- Замір часу виконання для обох підходів.
- Перевірка коректності введених даних користувачем.

## Використання

1. Запустіть програму.
2. Введіть розміри матриць та діапазон значень елементів.
3. Програма виведе згенеровані матриці, результат множення та час виконання для кожного підходу.

## Порівняння підходів

Програма заміряє час виконання для обох підходів: **Work Stealing** та **Work Dealing**. Час виконання для кожного з підходів виводиться на екран для порівняння ефективності.

### Очікувані результати

- **Work Stealing**: Може бути ефективнішим для задач із великою кількістю підзадач завдяки динамічному перерозподілу роботи між потоками.
- **Work Dealing**: Підхід, при якому кожен потік виконує свою частину роботи без перерозподілу. Це може бути корисно для задач, де частини роботи є більш рівномірними.

## Технічні деталі

- Для реалізації використано:
  - `ForkJoinPool` для підходу **Work Stealing**.
  - `ExecutorService` з пулом потоків для підходу **Work Dealing**.
  - Рандомна генерація матриць з заданими користувачем параметрами.
  - Перевірки вводу для коректності даних користувача.

## Висновки

Залежно від обсягу задачі, один з підходів може бути швидшим за інший. Рекомендується використовувати **Work Stealing** для великих задач з великими матрицями, де є необхідність у динамічному перерозподілу робочих навантажень.
